## Proxy pattern
- Proxy는 대리인 이라는 뜻으로써, 뭔가를 대신해서 처리하는 것
- Proxy Class를 통해서 대신 전달하는 형태로 설계되며, 실제 Client는 Proxy로 부터 결과를 받는다.
- Cache의 기능으로도 활용이 가능하다.
- SOLID중에서 개방폐쇄 원칙(OCP)과 의존역전 원칙(DIP)를 따른다.
